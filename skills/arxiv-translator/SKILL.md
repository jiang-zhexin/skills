# arXiv 论文中文翻译

**目标：** 将指定论文的 LaTeX 源码译为中文，并编译得到 PDF。

**流程：** 须严格按下文「第一步」至「第三步」顺序执行，不得擅自省略、合并或调换步骤。

**交互：** 仅在论文 ID 无法确定、检索结果存在多个需用户择一才可向用户提问；其余情况一律无中断的执行得到最终翻译后的PDF。

**翻译：** 翻译全部由当前对话模型自身完成，严禁使用外部翻译工具以及下载已有的翻译版本。

## 第一步：下载 latex 源码

使用以下脚本下载 latex 源码。

```bash
uv run scripts/download.py <arxiv_id>
```

脚本最终输出 latex 下载完成的目录和目录树：

```plaintxt
success download: {OUTPUT_DIR}
{the dir tree}
```

## 第二步：翻译

由当前**对话模型**在 `.tex` 文件上进行翻译。必须根据项目规模选择策略——**严禁一次性读取全文翻译**。

### 2.0 通用翻译规则

以下规则适用于所有翻译任务：

- **翻译范围：** 默认只翻正文，不翻附录，但附录中的内容需要得到保留，若同一文件中出现 `\appendix`，默认只翻该命令之前的内容。用户明确要求"翻译全文"时才翻附录。
- **必须翻译：** 正文叙述、摘要、图表标题、列表项、脚注中的描述文本，以及代码块中的注释。
- **保留不翻：** 数学环境、LaTeX 命令、`\cite{}`/`\ref{}`/`\label{}`、图片路径、URL、代码本体、`.bib`、人名、机构名、模型名、数据集名。
- **专有名词：** Transformer、Softmax、Token 等通用学术术语保留英文，不要生硬硬译。
- **标题要求：** `\title{}` 须改为自然中文题名，不保留英文原题或中英并列。
- **保持编译完整性：** 每次编辑后，务必保证 LaTeX 语法完整（括号配对、环境闭合、命令参数齐全）。若不确定某段是否影响编译，宁可保留原文不翻。

### 2.1 策略选择

根据第一步的分析结果，按以下规则选择策略：

| 条件                          | 策略                                              |
| ----------------------------- | ------------------------------------------------- |
| 仅 1 个 .tex 文件 且 < 400 行 | **策略 A：直接翻译** — 在主对话中一次性读取并翻译 |
| 仅 1 个 .tex 文件 且 ≥ 400 行 | **策略 B：分块翻译** — 见 §2.2                    |
| 多个 .tex 文件                | **策略 C：并行委派** — 见 §2.3                    |

### 2.2 策略 B：分块翻译（长文件）

当单个文件过长时，必须分块处理：

1. **分块方式：** 以 `\section{...}` 为自然分界点，每块包含 1-2 个 section（约 100-350 行）。
2. **逐块流程：**
   a. 用 Read 读取当前块的原文（读取范围精确到一个或多个完整 section）。
   b. 直接翻译该块内容，遵循 §2.0 所有规则。
   c. 用 Edit 将译文写回原文件对应位置（精确匹配原文，替换为译文）。
   d. 继续下一块，直到 `\appendix` 或文件末尾。
3. **关键约束：**
   - 每块结束前必须确认块内的 `\begin{...}` 与 `\end{...}` 全部配对闭合。
   - 严禁一次性读取超过 500 行的内容做翻译——超过即切分。
   - 跨 section 的浮动体（`\begin{figure}...\end{figure}`、`\begin{table}...\end{table}`）放在其所在的 section 块中翻译，不要跨块切割。

### 2.3 策略 C：并行委派（多文件）

当存在多个 `.tex` 文件时使用 Subagent 并行翻译：

1. **主文件：** 含 `\documentclass` 的文件通常最大，留在主对话中按策略 A 或 B 处理。
2. **子文件委派：** 其余每个子文件创建一个 Task subagent，每个子文件一个 subagent。
3. **Subagent prompt 模板：**

```
你是学术论文翻译助手。请翻译以下 LaTeX 文件为中文。

文件路径：<child_file_path>

术语表（必须保持一致）：
<从摘要/引言中提取的关键术语中英对照>

翻译规则：
- 只翻正文叙述、摘要、图表标题、列表项、脚注、代码注释
- 保留不翻：数学环境、LaTeX 命令、\cite/\ref/\label、图片路径、URL、代码、人名、机构名、模型名、数据集名
- 专有名词保留英文原文
- 文件中间的 \appendix 之前的内容翻译，之后只保留不翻
- 保持 LaTeX 语法完整，括号配对、环境闭合
- 直接在原文件上用 Edit 修改

完成后报告：翻译了哪些 section，遇到什么问题。
```

4. **术语表：** 在委派前，主对话从主文件的 Abstract/Introduction 部分提取 10-20 个关键术语及其中文翻译，写入每个 subagent 的 prompt 中，确保全篇术语一致。
5. 所有 subagent 完成后，主对话检查各文件间的交叉引用是否仍然有效。

## 第三步：编译与修复

编译 latex 使用 tectonic。

> Tectonic is a modernized, complete, self-contained TeX/LaTeX engine, powered by XeTeX and TeXLive.

如果用户未安装 tectonic，则引导用户到 https://tectonic-typesetting.github.io/en-US/install.html 进行安装后再进行编译。

[完整命令行参数](references/tectonic.md)。

```bash
tectonic <main_file_path>
```

### 3.1 编译失败处理

若编译报错，按以下流程修复：

1. 读取错误信息，定位报错的文件和大致行号。
2. 常见错误：
   - `Undefined control sequence`：检查是否误翻了 LaTeX 命令名
   - `Runaway argument`：检查括号 `{}` 是否配对
   - `Missing \begin{document}`：检查是否误改 `\documentclass` 或导言区
   - `Missing } inserted`：环境未闭合或 `\textit{...}` 等命令缺少 `}`
3. 修正后重新编译，最多尝试 3 次。若仍失败，向用户报告具体错误。

编译完成后输出 PDF 保存路径。
